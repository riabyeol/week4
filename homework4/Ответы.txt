1. CSS стили можно подключить записав код напрямую в тег <style> и поместив его в тег <head>(такой способ не самый удобный для больших работ). Второй вариант - это создать отдельный файл style.css и записать всю нужную информацию туда. Затем просто добавить ссылку на css файл в тег <head>  html документа. третий способ подключения стилей - это просто вписывать атрибут style и нужные нам параметры прямо в конкретный тег. Минусы такого способа в том, что изменение стиля будет происходить только в одном теге, не затрагивая другие теги того же сеемейства.
2. normalize.css нужен для обсепчивания лучшей кроссбраузности html элементов в стилях по умолчанию.

4. margin создает отступы относительно внешних границ объекта к соседним блокам. 
padding создает отступы между самим объектом и его внутреней границей.
5. Приоритеты в CSS путем вычисления специфичности селекторов. Например, идентификаторы стоят 100; классы - 10; селекторы - 1. Чем больше сумма селекторов, тем выше приоритет.
Также в ситуации когда в коде есть два одинаковых селектора, последний их них будет являться приоритетным.
Между селекторами  #link, .main, span #login, более приоритетным будет считаться - span #login.
6. CSS - это какадная таблица стилей, которая используется для создания дизайна для веб-страниц.  CSS3 - это улучшенная версия CSS. Она используется для структурирования, стлизации и форматирования веб-страниц, а также позволяет нам более комфортно и эффективно создавать веб-дизайн страницы.
7. Псевдоклассы - это дополнительные сведения, не указанные в документе, но добавленные браузером самостоятельно. Псевдоклаасы позволяют стилизовать элементы, анализируя разметку и историю пользователя. 
Псевдоэлементы создают так называемые "виртуальные теги", или по-другому ключевые слова, добавляемые к селектору  и позволяют стилизовать их.
8. Список "плохих" тегов:
<big> увеличивает размер текста. Лучше использовать CSS font-size, либо если жто заголовок, соответсвующие теги <h1>,<h2> и т.д.
<small> уменьшает размер текста. Лучше использовать CSS font-size.
<u> подчеркивает текст. Часто подчеркнутый текст ассоциируется с ссылками, поэтому тег <u>  уже давно не используется.
<b> отображает текст жирным шрифтом, вместо него лучше использовать тег <strong>, который усиливает выделение и акцентируется при прочтении текста голосовым помошником, что делает текст более доступным и понятным для людей с ограниченными возможностями. Либо выделить текст также можно в  CSS.
Такдже тег <i> отображающий текст курсивом, вместо него лучше использовать тег <em>, так как он добавляет доболнительную информацию выделения.
<layer> старый аналог тега <div>, работающий только в старых браузерах и потому абсолютно бесполезен.
9. Для локального подключения шрифтов нужно загрузить желаемые файлы шрифтов к себе на компьютер (желательно сразу в нескольких форматах, это поможет для кроссбраузности). Добавить их в папку для удобства использования. Например, в папку fonts, которая находится внутри папки assets. Затем нужно создать директив @font-face в CSS файле и задать нужные нам настроики шрифта.
10. Ипользование сокращённой записи в шрифтах (и не только) плохо тем что, если какой-то селектор не был указан, то он будет прописан по умолчанию. Либо наоборот, если прописать, например в сокращенной записи жирность шрифту, а затем отдельно указать, что текст должен быть прописан курсивом, то сокращеная версия будет являться доминантной и текст проявиться жирным. 
Иногда, для удобства и упрощения кода, можно использовать сокращенную запись, но нужно учитывать ньюансы.
